# 전환율 벤치마크

랜딩 → 가입 → 유료 전환까지의 각 단계별 기준. "내 전환율이 잘 되고 있는가" 판단용.

## 퍼널 전환율 (업종별 평균, 2025년 기준)

### 랜딩페이지 전환율 (방문 → 원하는 행동)

| 유형 | 평균 | 좋은 수준 | 탁월한 수준 | 행동 정의 |
|------|------|-----------|-------------|-----------|
| 이메일 수집 | 5-8% | 10-15% | 20%+ | 이메일 입력 |
| 무료 체험 가입 | 3-5% | 7-10% | 15%+ | 회원가입 |
| 유료 구매 (저가) | 1-3% | 3-5% | 7%+ | ₩10,000 이하 결제 |
| 유료 구매 (고가) | 0.5-1% | 1-3% | 5%+ | ₩50,000+ 결제 |
| 앱 다운로드 | 5-10% | 15-20% | 30%+ | 설치 |
| 상담 신청 | 2-5% | 5-10% | 15%+ | 폼 제출 |

### 폼 응답률 (폼 노출 → 제출)

| 폼 유형 | 평균 | 좋은 수준 | 비고 |
|---------|------|-----------|------|
| 이메일만 | 10-15% | 20-30% | 필드 1개 |
| 이름 + 이메일 | 5-10% | 15-20% | 필드 2개 |
| 상세 폼 (3-5필드) | 3-5% | 7-10% | 필드 늘수록 하락 |
| 설문 (10+문항) | 1-3% | 5-7% | 인센티브 있으면 2-3x |

**법칙**: 폼 필드 1개 추가 시 전환율 ~7-10% 하락.

### SaaS 전환 퍼널

| 단계 | 평균 전환율 | 좋은 수준 | 탁월한 수준 |
|------|-----------|-----------|-------------|
| 방문 → 가입 | 2-5% | 5-10% | 10-15% |
| 가입 → 활성화 (Aha moment) | 20-40% | 40-60% | 60%+ |
| 활성화 → 유료 전환 | 3-5% | 5-10% | 10-15% |
| 유료 → 1개월 유지 | 85-90% | 90-95% | 95%+ |
| 월간 이탈률 (Churn) | 5-10% | 3-5% | 2% 이하 |

### 이메일 마케팅

| 지표 | 평균 | 좋은 수준 | 비고 |
|------|------|-----------|------|
| 오픈율 | 20-25% | 30-40% | 업종/리스트 크기별 차이 |
| 클릭률 (CTR) | 2-3% | 4-6% | 오픈 대비 |
| 구독 해지율 | 0.2-0.5% | 0.1% 이하 | 발송당 |

## 1인 개발자 제품별 기준

### 디지털 콘텐츠 (템플릿, 가이드, PDF)

```
랜딩 방문 → 구매: 2-5% (좋으면 5-10%)
소셜 포스트 → 클릭: 1-3%
```

100명 방문에 3-5명 구매. ₩10,000 제품이면 ₩30,000-50,000 매출.

### SaaS / 구독 서비스

```
랜딩 방문 → 무료 가입: 3-7%
무료 가입 → 유료 전환: 3-5% (좋으면 5-10%)
유료 → 월간 유지: 90%+
```

1,000명 방문 → 50명 가입 → 2-5명 유료. 월 ₩10,000이면 ₩20,000-50,000 MRR.

### 커뮤니티 / 코호트

```
랜딩 방문 → 신청: 5-10%
신청 → 결제: 20-40% (가격에 따라)
```

## 성과 판단 가이드

### 빠른 진단

```
전환율 < 1% → 문제 있음 (오퍼 또는 랜딩 재검토)
전환율 1-3% → 보통 (최적화 여지 있음)
전환율 3-5% → 좋음 (스케일 가능)
전환율 5%+ → 탁월함 (트래픽 늘려라)
```

### 전환율이 낮을 때 점검 순서

1. **오퍼** — "이게 돈 주고 살 만한 가치가 있나?" (가격, 제공 가치)
2. **헤드라인** — 방문 후 3초 안에 "이건 나를 위한 거다" 확신이 드나?
3. **CTA** — 버튼이 명확하고 눈에 띄나? (하나의 행동만)
4. **소셜 프루프** — 후기, 사용 사례, 숫자가 있나?
5. **속도** — 3초 안에 로딩되나? (모바일 기준)

### 표본 크기 주의

- 100명 미만 방문에서 전환율 판단하지 마라
- 최소 200-300명 방문 데이터 필요
- 통계적 유의성: A/B 테스트 시 최소 400명/변형
