## Agentic30 맥락에서

30일 안에 현실적으로 달성 가능한 것:

| 목표 | 30일 달성 가능성 | 조건 |
|------|-----------------|------|
| 문제 정의 + ICP | ✅ 높음 | 1주 |
| 인터뷰 3회 | ✅ 높음 | 2주 |
| SPEC 작성 | ✅ 높음 | 3주차 |
| 랜딩페이지 배포 | ✅ 높음 | 3주차 |
| 첫 트래픽 유입 | ✅ 가능 | 채널 1-2개 활성화 |
| 폼 응답 수집 | ⚠️ 가능 | 100+ 방문 필요 |
| 유료 시그널 (결제 의향) | ⚠️ 도전적 | 오퍼 + 채널 + 운 |
| 실제 첫 매출 | ❌ 어려움 | 대부분 2-4개월 |

**"30일 안에 첫 유료 시그널"의 의미**: 실제 매출이 아니라, 누군가 돈을 낼 의향을 보인 것 (폼 응답, 결제 페이지 도달, 가격 문의). 이것만으로도 상위 10% 속도.

## 성과 판단

```
30일 내 인터뷰 3회 + SPEC + 랜딩 완료 → 상위 20% 속도
30일 내 트래픽 100+ + 폼 응답 5+ → 상위 10% 속도
30일 내 유료 시그널 1개 → 상위 5% (매우 빠름)
30일 내 첫 매출 → 상위 1% (예외적)
```

느리다고 실패가 아니다. 대부분의 성공한 인디해커는 첫 매출까지 2-6개월 걸렸다. 핵심은 **멈추지 않는 것**.
