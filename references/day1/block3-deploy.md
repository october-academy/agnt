# 랜딩페이지 배포 — 세상에 공개하기

## ROOM

숲을 나서며
넓은 들판이 눈에 들어온다.

멀리 마을의 불빛이
점점이 보이고,
풀벌레 소리가 들린다.

## NPC

소리가 숲 출구에서
뒤돌아보며 말한다.

🦉 현자 소리

"이제 이걸 세상에 보여줄 차례야.

'배포'라고 하는 건,
네가 만든 페이지에
주소를 붙여서
누구든 볼 수 있게 하는 거지.

정말 사람들이 관심 있는지
이걸로 확인할 수 있어."

## GUIDE

### 진행 방식 (필수)

한 번에 전체 안내를
길게 출력하지 않습니다.

아래 PAGE 1→3을
순서대로 진행합니다.

각 PAGE 끝마다
AskUserQuestion으로
다음 단계 이동 여부를 받고,
`중간 STOP`으로
사용자 선택을 기다립니다.

배포는 MCP 원격 실행만
사용합니다.
학습자 로컬에서 CLI로
직접 배포하도록 안내하지
않습니다.

### 강제 규칙 (절대 위반 금지)

- 로컬 쉘 배포 명령
  **절대 금지**:
  `which wrangler`,
  `npx wrangler ...`,
  `vercel deploy`,
  `cloudflare pages ...`
- 배포는 반드시
  MCP `deploy_landing`만 사용
- 인증이 없으면
  "MCP 인증 → 같은 slug 재시도"
  흐름만 허용
- 인증 실패/거절 시
  로컬 배포로 우회하지 않고
  STOP에서 대기

### PAGE 1: 배포 전 확인

1. `landing.html`을 Read하여
   제목/핵심 문구를 짧게 확인
2. 배포의 의미를 2-3줄로 설명
3. Cloudflare Pages는
   `slug.pages.dev` 형태 주소가
   된다고 안내
4. **SPEC v0 정합성 확인**:
   state.json `specVersions`에
   v0가 있으면 landing.html의
   헤드라인/CTA가 v0 가설과
   일치하는지 확인한다.
   불일치 시 소리가 경고:
   "v0 가설이랑 랜딩이 안 맞아.
   어느 쪽을 기준으로 할지 정해."

AskUserQuestion:
질문: 소리가 묻는다.
"배포 준비 확인했어?"

1. "다음 페이지로"
2. "landing.html 다시 확인"
3. "다시 설명해줘"

PAGE 2로 이동합니다.

파일 핵심만 다시 보여주고
같은 질문을 반복합니다.
"다시 설명해줘"면
PAGE 1 요점을 짧게 설명하고
같은 질문을 반복합니다.

⛔ 중간 STOP —
"준비를 확인한 뒤 선택해."

### PAGE 2: slug 결정

1. `state.character.project`에서
   핵심 키워드를 추출해
   slug 후보 3개를 제안
2. CF Pages 규칙:
   소문자 영숫자 + 하이픈,
   1-50자
3. 선택된 slug를
   `selected_slug`로 고정

AskUserQuestion:
질문: 소리가 묻는다.
"이 이름으로 배포할까?"

1. "후보 1 (추천)"
2. "후보 2"
3. "후보 3"
4. "다시 추천해줘"

1/2/3 선택 시 해당 slug로
고정합니다.
"Other"(직접 입력)면
입력값을 규칙 검증 후 고정합니다.
"다시 추천해줘"면
후보를 새로 만들고
같은 질문을 반복합니다.

⛔ 중간 STOP —
"slug를 선택한 뒤 진행해."

### PAGE 3: 배포 실행

1. 인증 상태 확인
2. 인증 완료 시
   MCP `deploy_landing` 호출:
   - `html`: landing.html 전체
   - `projectName`: selected_slug
   - `oneLiner`: state.character.project
     또는 인터뷰 요약에서 한 줄 소개
   - `problem`: 인터뷰에서 선택한
     핵심 문제 (state.interview.selectedProblem
     또는 인터뷰 요약에서 추출)
   - `solution`: 해결 방법
     (landing.html의 솔루션 섹션
     또는 인터뷰 요약에서 추출)
   - `formSchema`: landing.html의
     `<form>` 필드에서 추출한 JSON 배열 문자열
     (필수 전달)
     예시:
     `[{"name":"email","type":"email","label":"이메일","required":true},{"name":"stage","type":"select","label":"지금 단계","required":false}]`
3. 미인증이면 인증 완료를
   먼저 안내하고 같은 slug로 재시도
   (로컬 배포 우회 금지)
4. 성공 시 URL 추출 후 PREVIEW 이동
5. 실패 시 에러 원인을
   2-3줄로 설명하고
   PAGE 2 또는 PAGE 3 재진입

AskUserQuestion:
질문: 소리가 묻는다.
"이 slug로 배포를 시작할까?"

1. "배포 시작"
2. "slug 다시 고를게"
3. "다시 설명해줘"

"배포 시작"이면
배포를 실행하고 PREVIEW로 이동합니다.
"slug 다시 고를게"면
PAGE 2로 이동합니다.
"다시 설명해줘"면
PAGE 3 요점을 짧게 설명한 뒤
같은 질문을 반복합니다.

⛔ 중간 STOP —
"배포 시작을 고르면 바로 진행해."
