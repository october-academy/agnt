# 목표 선언 — 무엇을 만들지 최종 확정

## ROOM

광장 안쪽 서약의 돌 앞에 선다.
돌 표면에는 이전 참가자들의
짧은 선언이 빼곡하다.

## NPC

두리가 돌 앞에서 돌아선다.

🏘️ 촌장 두리

"두 가지를 고정한다.

1. 이번에 밀 프로젝트
2. 30일 목표

멋진 문장 말고
검증 가능한 문장."

## CONVERSATION

Day 1 검증 루프의 출발점을 확정합니다.

### 완성 체크리스트

아이디어 없음 / 아이디어만 있음
- [ ] 후보 2개 이상 비교
- [ ] 최종 프로젝트 1개 확정
- [ ] 왜 이걸 먼저 미는지 이유 3가지
- [ ] 30일 목표 1개
- [ ] 24시간 안 첫 검증 행동 1개

랜딩 운영 중 / 제품 운영 중 / 매출 발생 중
- [ ] 현재 단계 확정
- [ ] 가장 큰 병목 1개
- [ ] 30일 목표를 현재 단계에 맞게 재선언
- [ ] 이번 주 검증 목표 1개
- [ ] 24시간 안 첫 행동 1개

### 목표 카테고리 분류

대화 내용에서 참가 목적을 추론합니다.
5개 중 하나 (DB 값 그대로 사용):
- startup (창업 / 전업 개발)
- side_income (부수입 목적)
- ai_learning (AI/기술 학습)
- employment (취업 준비)
- job_change (이직/전직)

**카테고리별 NPC 프레이밍 (두리 대사)**:

- `startup`: 두리가 먼저 권유한다.
  "이 프로그램은 '만들고 파는 사람'에 최적화돼 있어.
  전업으로 밀고 있다면 여기가 맞아."
  → 1순위로 배치, NPC가 적극 권유.

- `side_income`: 원래 라벨 유지.
  "사이드 수익으로 시작해도 돼.

  한 블록에 30분이면 되거든.
  '언제 할지'를 고정해.

  속도가 아니라 리듬이 중요해."

- `ai_learning`: 원래 라벨 유지.
  "배우기만 하면 끝이 없어.
  배우면서 만들어."

- `employment` / `job_change`: 하단 배치.
  "이 프로그램은 '만들고 파는 사람'에 최적화돼 있어.
  그래도 괜찮다면 환영해."

SUMMARY에서 카테고리를 포함하고,
유저에게 맞는지 확인합니다.

### 런웨이 확인

두리가 목표를 적기 전에 묻는다.

"좋아. 마지막으로 하나만.
지금 저축으로 몇 개월 버틸 수 있어?
본업이 있으면 하루에 몇 시간 쓸 수 있어?

이건 목표 난이도를 맞추려는 거야.
네 속도에 맞게 검증 루프를 짤 거거든."

응답을 `character.runway` (월 단위)로 저장합니다.
- 저축 N개월 → `runway: N`
- 하루 M시간 가용 (본업 있음) → `runway: 12` (사이드 프로젝트 기본값)
- 답변 거부 → `runway: null` (이후 DIALOGUE_GATE에서 미존재 시 fallback 사용)

### 목표 선언 기준

- "성공" 같은 추상어 금지
- 행동/숫자/증거가 들어가야 함
- Agentic30 기본 미션과 충돌하지 않아야 함
  - 실제 사용자 반응
  - 첫 수익화 신호
- 두리가 확인: "30일 뒤 저축은 줄어든다.
  그 전에 뭘 확인할 거야?"

예시:

- "나는 30일 안에 freelancer PM 10명에게서
  반복 문제 증거를 확보한다."
- "나는 30일 안에 현재 랜딩에서
  유료 시범 제안으로 구체적 결제 의향 1건을 만든다."

## SUMMARY

두리가 돌 표면을 손가락으로 짚는다.

```text
최종 선언
- 프로젝트/병목: ...
- 왜 지금 이걸 미는가: ...
- 목표 카테고리: ...
- 30일 목표: ...
- 이번 주 검증 목표: ...
- 24시간 액션: ...
```

두리:
"좋아. 내일 SPEC baseline의 입구야."

## STOP

AskUserQuestion:
질문: 두리가 묻는다. "목표 선언문이
맞아?"

1. "확인"
2. "수정 요청"

⛔ STOP — "촌장 두리가 기다립니다."

## ON_COMPLETE

AskUserQuestion에서
"확인"을 선택했을 때만
ON_COMPLETE를 수행합니다.
"수정 요청"이면
CONVERSATION으로 돌아가
보완 후 SUMMARY → STOP을 반복합니다.

1. state.json에
   `character.project`, `character.target`,
   `goal`, `nextAction`, `nextProofTarget`,
   `character.runway` (월 단위 정수 또는 null)을 저장합니다.
2. 이미 뭔가 만든 사람은
   새 프로젝트 대신 현재 병목과 검증 목표를 저장합니다.
3. 대화에서 추론한 목표 카테고리를
   state.json `character.goalCategory`에 저장합니다.
4. 인증 상태면
   `save_profile`에 `goalCategory` 파라미터를 포함하여
   `save_profile`과 `submit_practice`
   흐름을 수행합니다.
5. goalCategory가 `startup`일 경우에만
   state.json `character.isFullTime`에 `true`를 저장합니다.
   (나머지 카테고리에서는 이 필드를 저장하지 않습니다.)

## MOVE

두리가 골목 쪽을 가리킨다.

"좋아. 이제 동료들한테
보여주자."
