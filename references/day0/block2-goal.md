# 목표 선언 — 무엇을 만들지 최종 확정

## ROOM

광장 안쪽 서약의 돌 앞에 선다.
돌 표면에는 이전 참가자들의
짧은 선언이 빼곡하다.

## NPC

두리가 돌 앞에서 돌아선다.

🏘️ 촌장 두리

"여기선 두 가지를 고정한다.

1. 이번에 밀 프로젝트 또는 병목
2. 30일 목표

내일부터는
그 선언을 기준으로
builder 상태와 검증 결과를 읽을 거야.

그러니까 멋진 문장보다
검증 가능한 문장이 필요해."

## CONVERSATION

이 블록의 목적은
Day 1부터 시작할 검증 순환 —
가설을 세우고, 검증 결과를 모으고, 판단하고, 다시 가설을 세우는 루프 —
의 출발점을 명확히 하는 것입니다.
여기선 이 순환을 `검증 루프`라고 부릅니다.

### 완성 체크리스트

아이디어 없음 / 아이디어만 있음
- [ ] 후보 2개 이상 비교
- [ ] 최종 프로젝트 1개 확정
- [ ] 왜 이걸 먼저 미는지 이유 3가지
- [ ] 30일 목표 1개
- [ ] 24시간 안 첫 검증 행동 1개

랜딩 운영 중 / 제품 운영 중 / 매출 발생 중
- [ ] 현재 단계 확정
- [ ] 가장 큰 병목 1개
- [ ] 30일 목표를 현재 단계에 맞게 재선언
- [ ] 이번 주 검증 목표 1개
- [ ] 24시간 안 첫 행동 1개

### 목표 카테고리 분류

대화 내용에서 참가 목적을 추론합니다.
5개 중 하나:
- startup (창업 준비)
- side_income (부수입/사이드 프로젝트)
- ai_learning (AI/기술 학습)
- employment (취업 준비)
- job_change (이직/전직)

SUMMARY에서 카테고리를 포함하고,
유저에게 맞는지 확인합니다.

### 목표 선언 기준

- "성공" 같은 추상어 금지
- 행동/숫자/증거가 들어가야 함
- Agentic30 기본 미션과 충돌하지 않아야 함
  - 실제 사용자 반응
  - 첫 수익화 신호

예시:

- "나는 30일 안에 freelancer PM 10명에게서
  반복 문제 증거를 확보한다."
- "나는 30일 안에 현재 랜딩에서
  유료 시범 제안으로 구체적 결제 의향 1건을 만든다."

## SUMMARY

두리가 돌 표면을 손가락으로 짚는다.

```text
최종 선언
- 프로젝트/병목: ...
- 왜 지금 이걸 미는가: ...
- 목표 카테고리: ...
- 30일 목표: ...
- 이번 주 검증 목표: ...
- 24시간 액션: ...
```

두리:
"좋아.
이 문장은 내일 `SPEC v0`의
입구가 될 거야."

## STOP

AskUserQuestion:
질문: 두리가 묻는다. "목표 선언문이
맞아?"

1. "확인"
2. "수정 요청"

⛔ STOP — "촌장 두리가 기다립니다."

## ON_COMPLETE

AskUserQuestion에서
"확인"을 선택했을 때만
ON_COMPLETE를 수행합니다.
"수정 요청"이면
CONVERSATION으로 돌아가
보완 후 SUMMARY → STOP을 반복합니다.

1. state.json에
   `character.project`, `character.target`,
   `goal`, `nextAction`, `nextProofTarget`을 저장합니다.
2. 이미 뭔가 만든 사람은
   새 프로젝트 대신 현재 병목과 검증 목표를 저장합니다.
3. 대화에서 추론한 목표 카테고리를
   state.json `character.goalCategory`에 저장합니다.
4. 인증 상태면
   `save_profile`에 `goalCategory` 파라미터를 포함하여
   `save_profile`과 `submit_practice`
   흐름을 수행합니다.

## MOVE

두리가 골목 쪽을 가리킨다.

"좋아.
이제 동료들한테도
지금 어디에 서 있는지
보여주자.

혼자 끌고 가지 말고."
