# 한국 솔로프리너 실무 가이드

한국에서 1인 개발자/인디해커로 프로덕트를 만들고 매출을 내기까지 알아야 할 현실적인 가이드.

---

## 한국에서 매출을 내는 현실

미국에서는 개인 자격으로 Stripe, 앱스토어, 플레이스토어 결제를 받을 수 있다. 소득이 발생하면 다음 년도에 세금 신고만 하면 된다. 별도 사업자 등록 없이, 소득이 없으면 페널티도 없다.

**한국은 다르다.** PG사를 통한 카드결제는 사업자등록이 필요하고, 등록 시 여러 부담이 따른다. 다만 프리랜서 용역 대금(3.3% 원천징수)이나 간편결제 앱을 통한 소규모 거래는 사업자 없이도 합법적으로 가능하다.

### 사업자등록 시 주의할 점

1. **건강보험** 피부양자 탈락 가능 → 지역가입자 전환 시 최저 월 19,500원 (1년차 80% 감면 제도 있음)
2. **국민연금** 지역가입자 가입 대상 — 단, 소득 없으면 납부예외 신청으로 0원 가능
3. **부모님 회사 가족수당** 자격 상실 가능 (회사 취업규칙에 따라 다름)
4. **기장 의무** — 신규사업자는 간편장부(가계부 수준)로 충분, 세무사 필수 아님
5. **병특 복무 중** 겸직 금지 규정에 저촉될 수 있음
6. **학자금 대출** 사업소득 발생 시 소득구간 변경 가능 (소득 없으면 영향 없음)
7. **예비창업패키지** 지원 자격 즉시 상실 (공고 기준 명시)

회사 재직 중이라면 겸업금지 조항 확인 필요.

**결론: 사업자등록은 매출이 확인된 후. 먼저 사업자 없이 검증부터.**

## 사업자 없이 결제 받는 방법

### 간편결제 앱 (국내, 사업자 불필요)
- **PayApp** — 개인도 사용 가능 (연 2,400만원 한도, 카드 수수료 4%)
- **랫피드** (latpeed.com) — 크리에이터/디지털 콘텐츠 결제 (수수료 ~1.6%)
- **딜앱** (dealapp.co.kr) — 소규모 거래 (연 5,000만원 한도)

### MoR 서비스 (해외 결제)
- **Lemon Squeezy** — 사업자 없이 해외 결제 수취. 디지털 상품/SaaS에 적합.
- **Polar** — 오픈소스/개발자 도구 수익화

MoR(Merchant of Record)은 판매자 대신 결제를 처리하고 세금 신고까지 대행한다. 한국 사업자 없이 글로벌 매출을 낼 수 있는 방법.

### 광고 수익
- **AdMob** — 앱 내 광고
- **AdSense** — 웹사이트 광고
- 소규모 수익은 사업자 없이 가능하나, 지속적·반복적 광고 수익은 사업자등록 의무 발생 가능 (국세청 기준).

## Build in Public (BIP) 채널

한국 인디해커의 주요 BIP 채널:
- **X (구 Twitter)** — 글로벌 인디해커 커뮤니티의 메인 채널. Premium 계정 시 도달 10배.
- **Threads** — 한국 사용자 활발. 체류 시간 기반 알고리즘.

BIP는 만드는 과정을 공개적으로 공유하는 전략. 완성된 제품이 아니어도 된다. 과정 자체가 콘텐츠.

---

## 한국 인디해커 커뮤니티

- **GeekNews** (news.hada.io) — 한국 최대 테크 커뮤니티. Show 포스트로 프로젝트 공유. 가입 후 7일 대기.
- **EOPlanet** (eopla.net) — 스타트업 아티클 커뮤니티. 빌드 스토리 형식이 맞음.
- **OKKY** (okky.kr) — 개발자 커뮤니티. 피드백 게시판에서 사이드프로젝트 공유.

상세 채널 가이드: `promotion-channels-guide.md` 참조.

## Cold Call / DM 검증

시장 반응을 검증하는 가장 직접적인 방법:
- 타겟 유저에게 직접 DM을 보낸다.
- "이런 서비스를 준비 중인데 의견 주시면 감사하겠습니다" 형식.
- 10명에게 보내면 2-3명이 답장한다. 숫자 게임.
- 칭찬이 아니라 솔직한 반응을 유도해야 한다 (Mom Test 원칙).

## 사업자 등록 결정 시

매출 검증이 끝나고 사업자를 내기로 결정했다면:

### PG사 선택
- **토스페이먼츠** — 개발자 친화적, 문서 우수
- **포트원** (구 아임포트) — 통합결제, 여러 PG사 연동
- **카카오페이** — 간편결제, 한국 유저 친숙

### 세금 기초
- **간이과세자**: 연 매출 8,000만원 이하, 부가세 간편 신고
- **일반과세자**: 매출 규모 커지면 전환 필요
- **종합소득세**: 매년 5월 신고 (사업소득 + 기타소득)
- **부가세**: 1월/7월 신고 (간이과세자는 연 1회)

### 법률
- **개인정보처리방침**: 개인정보 수집 시 필수
- **이용약관**: 서비스 이용 조건 명시
- **통신판매업 신고**: 온라인 판매 시 필수 (정부24에서 온라인 신고 가능)

## 첫 매출까지 타임라인

사이드프로젝트에서 첫 매출까지의 현실적 경로:

### Phase 1: 검증 (Week 1-2) — 사업자 불필요
1. 문제 발견 + SPEC 작성
2. 랜딩페이지 배포 + UTM 추적 시작
3. 한국 채널(GeekNews, OKKY, BIP)에서 검증
4. 10명 이상 피드백 확보
5. **결정 포인트**: Go / Pivot / No-Go

### Phase 2: MVP 빌드 (Week 2-3) — 사업자 불필요
1. 핵심 기능만 빌드 (AI로 빠르게)
2. 얼리 엑세스 모집 (랜딩 폼 응답자 우선)
3. 사업자 없이 결제 테스트 (PayApp/랫피드 또는 Lemon Squeezy)

### Phase 3: 첫 매출 (Week 3-4) — 사업자 선택
1. 유료 전환 시도 (무료 → 유료 or 유료 직접 판매)
2. 첫 결제 발생
3. **사업자등록 결정 포인트**: 월 매출 50만원 이상 지속 시 고려

### 사업자등록 결정 시그널
- 월 매출 50만원 이상 3개월 연속
- PG사 직접 연동이 필요한 경우 (토스페이먼츠 등)
- 세금계산서 발행이 필요한 B2B 거래
- 정부 지원사업 신청 (사업자 필수)

**원칙: 매출이 먼저, 사업자등록은 나중.**

## 통신판매업 신고 체크리스트

온라인으로 상품/서비스를 판매할 때 필요:

- [ ] 사업자등록증 (간이과세자 가능)
- [ ] 정부24 → 통신판매업 신고 (온라인, 수수료 없음)
- [ ] 개인정보처리방침 게시 (개인정보보호위원회 템플릿 활용)
- [ ] 이용약관 게시 (서비스 이용 조건, 환불 정책 명시)
- [ ] 사업자정보 표시: 상호, 대표자, 사업자번호, 주소, 연락처, 통신판매업 신고번호

### 개인정보처리방침 필수 항목
1. 수집하는 개인정보 항목
2. 수집 목적
3. 보유 기간
4. 제3자 제공 여부
5. 파기 절차
6. 정보주체 권리

### 이용약관 필수 항목
1. 서비스 내용
2. 이용 요금 및 결제
3. 환불 정책 (전자상거래법: 7일 이내 청약철회)
4. 서비스 중단/변경 안내
5. 분쟁 해결
