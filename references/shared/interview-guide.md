# 심층 인터뷰 가이드

인터뷰 블록(Talk STOP)에서 참조하는 공통 원칙입니다.

## 핵심 원칙

### 1. Mom Test 준수

- **과거 경험** 기반 질문만 합니다: "마지막으로 그 문제를 겪었을 때..."
- **미래 의향** 질문 금지: "~하면 사용하시겠어요?" (거짓 긍정 유발)
- 칭찬/동의가 아닌 **구체적 행동**을 끌어냅니다

### 2. 응답 추적 (Follow the Thread)

- 사용자 답변에서 흥미로운 실마리를 포착하고 파고듭니다
- 표면적 답변에 만족하지 말고 "왜?" "구체적으로?" "그때 어떻게 했나요?"
- 예상 밖의 답변이 가장 귀중한 인사이트입니다

### 3. 반론 제기 (Challenge Assumptions)

- "~라고 가정하고 계신 것 같은데, 만약 ~라면요?"
- 사용자의 확신에 건설적으로 도전합니다
- 목적은 논쟁이 아니라 깊은 사고 유도

### 4. 침묵 활용

- 모든 질문에 선택지를 주되, "잘 모르겠어요"를 항상 포함합니다
- "잘 모르겠어요" 선택 시 다른 각도에서 접근합니다

## AskUserQuestion 포맷

```
AskUserQuestion:
질문: "{구체적이고 예측 불가능한 질문}"
선택지:
1. "{과거 경험 기반 답변 A}"
2. "{과거 경험 기반 답변 B}"
3. "{다른 관점의 답변 C}"
4. "잘 모르겠어요"
```

- 선택지 2-4개 + "잘 모르겠어요" 항상 포함
- 선택지는 Yes/No가 아닌 **구체적 시나리오**로 구성

## 커버리지 영역

인터뷰는 다음 영역을 모두 다뤄야 합니다:

- **문제/불편함**: 사용자가 실제로 겪는 문제 3개 이상
- **현재 해결법**: 지금 어떻게 해결하고 있는지
- **핵심 문제 1개**: 가장 시급하고 비용이 큰 문제
- **규모감**: 이 문제를 겪는 사람이 얼마나 되는지
- **형태**: 어떤 형태의 솔루션을 기대하는지
- **가격 시그널**: 현재 이 문제 해결에 쓰는 비용/시간

## Day 0 탐색 인터뷰 추가 규칙

Day 0(`block2-character`)에서는
"아이템 확정"이 아니라
"문제 탐색"이 목적입니다.

- **최소 8턴 미만 종료 금지**
- 문제 3개를 확보하기 전 SUMMARY 금지
- 후보 아이템 2-3개가 없으면 SUMMARY 금지
- 상위 후보 1개에 대해 **"왜?" 3연쇄** 필수
- 이 단계에서는 프로젝트명을 확정하지 않습니다
  (최종 확정은 Day 0 `block3-goal`)

## Day 1 검증 인터뷰 추가 규칙

Day 1(`block1-interview`)은
Day 0 탐색의 반복이 아니라
외부 검증 준비 단계입니다.

- Day 0에서 고른 핵심 문제를
  반론 관점에서 다시 검증
- 대안/경쟁/전환비용을 반드시 질문
- "누구에게 먼저 검증할지"를
  구체적 행동으로 마무리

## Few-shot 예시

### 좋은 질문

- "마지막으로 그 작업을 했을 때, 가장 시간이 많이 걸린 단계는 뭐였나요?"
- "지금 사용하는 도구에서 가장 짜증나는 점은 뭔가요? 구체적 상황을 알려주세요."
- "그 문제 때문에 실제로 포기하거나 다른 방법을 찾은 적 있나요?"
- "팀에서 이 문제를 어떻게 공유하나요? 아니면 개인적으로만 겪나요?"

### 나쁜 질문 (금지)

- "이 아이디어 어떻게 생각하세요?" → 사교적 동의 유발
- "이런 서비스가 있으면 쓰시겠어요?" → 미래 의향
- "얼마를 내실 의향이 있으세요?" → 직접적 가격 질문
- "그럼 다음 질문으로 넘어가볼까요?" → 인위적 전환
- "그럼 이걸로 만들까요?" → 탐색 단계 조기 확정

## 한국 문화 특수성

### 거절을 돌려말하는 문화

한국 사용자는 거절을 직접적으로 표현하지 않는 경향이 있다.

- "네" ≠ 동의. "생각해볼게요", "좋은 것 같아요"는 거절 신호일 수 있다
- **판별법**: "구체적으로 언제 써보실 건가요?", "이걸 위해 지금 뭘 하고 계세요?"로 행동 의지 확인
- 칭찬("대단하시네요", "잘 만드셨네요")은 관심이 아니라 예의일 수 있다

### 가격 민감도

- 한국 사용자에게 직접 가격을 묻지 않기 (Mom Test 원칙 강화)
- 대신: "이 문제를 해결하기 위해 지금 쓰고 있는 방법이 있나요?", "그 방법에 얼마나 시간/돈을 쓰고 있나요?"
- 한국 시장 가격대 참고: 개인용 SaaS 월 5,000~15,000원, 팀용 월 30,000~100,000원이 일반적

### 한국 검증 풀 찾기

| 채널 | 특성 | 응답률 |
|------|------|--------|
| OKKY 피드백 게시판 | 개발자 타겟 | 중간 |
| 카카오톡 오픈채팅 | 빠른 응답, 비공식적 | 높음 |
| GeekNews meetup/댓글 | 기술 깊이 있는 피드백 | 낮지만 질 높음 |
| LinkedIn DM | 직장인/전문직 타겟 | 낮음 |
| X/Threads DM | 인디해커 커뮤니티 | 중간 |

- LinkedIn/X보다 **카카오톡**이 한국에서는 더 높은 응답률
- 지인 네트워크 → 오픈채팅 → 커뮤니티 DM 순서로 확장

## 안티패턴

1. **빠른 종료**: 5턴 만에 "충분히 이해했습니다" — 절대 금지
2. **요약 확인**: "지금까지 정리하면..." 너무 자주 — 흐름 끊김
3. **유도 질문**: 원하는 답변으로 이끄는 질문
4. **체크리스트 읽기**: "규모감에 대해 물어볼게요" — 기계적
5. **칭찬 과다**: "정말 좋은 인사이트네요!" — 사교적 동의 유도
6. **조기 확정**: 후보 비교 없이 프로젝트를 바로 고정
