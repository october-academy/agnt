# 심층 인터뷰 가이드

인터뷰 블록(Talk STOP)에서 참조하는 공통 원칙입니다.

## 핵심 원칙

### 1. Mom Test 준수

- **과거 경험** 기반 질문만 합니다: "마지막으로 그 문제를 겪었을 때..."
- **미래 의향** 질문 금지: "~하면 사용하시겠어요?" (거짓 긍정 유발)
- 칭찬/동의가 아닌 **구체적 행동**을 끌어냅니다

### 2. 응답 추적 (Follow the Thread)

- 사용자 답변에서 흥미로운 실마리를 포착하고 파고듭니다
- 표면적 답변에 만족하지 말고 "왜?" "구체적으로?" "그때 어떻게 했나요?"
- 예상 밖의 답변이 가장 귀중한 인사이트입니다

### 3. 반론 제기 (Challenge Assumptions)

- "~라고 가정하고 계신 것 같은데, 만약 ~라면요?"
- 사용자의 확신에 건설적으로 도전합니다
- 목적은 논쟁이 아니라 깊은 사고 유도

### 4. 침묵 활용

- 모든 질문에 선택지를 주되, "잘 모르겠어요"를 항상 포함합니다
- "잘 모르겠어요" 선택 시 다른 각도에서 접근합니다

## AskUserQuestion 포맷

```
AskUserQuestion:
질문: "{구체적이고 예측 불가능한 질문}"
선택지:
1. "{과거 경험 기반 답변 A}"
2. "{과거 경험 기반 답변 B}"
3. "{다른 관점의 답변 C}"
4. "잘 모르겠어요"
```

- 선택지 2-4개 + "잘 모르겠어요" 항상 포함
- 선택지는 Yes/No가 아닌 **구체적 시나리오**로 구성

## 커버리지 영역

인터뷰는 다음 영역을 모두 다뤄야 합니다:

- **문제/불편함**: 사용자가 실제로 겪는 문제 3개 이상
- **현재 해결법**: 지금 어떻게 해결하고 있는지
- **핵심 문제 1개**: 가장 시급하고 비용이 큰 문제
- **규모감**: 이 문제를 겪는 사람이 얼마나 되는지
- **형태**: 어떤 형태의 솔루션을 기대하는지
- **가격 시그널**: 현재 이 문제 해결에 쓰는 비용/시간

## Few-shot 예시

### 좋은 질문

- "마지막으로 그 작업을 했을 때, 가장 시간이 많이 걸린 단계는 뭐였나요?"
- "지금 사용하는 도구에서 가장 짜증나는 점은 뭔가요? 구체적 상황을 알려주세요."
- "그 문제 때문에 실제로 포기하거나 다른 방법을 찾은 적 있나요?"
- "팀에서 이 문제를 어떻게 공유하나요? 아니면 개인적으로만 겪나요?"

### 나쁜 질문 (금지)

- "이 아이디어 어떻게 생각하세요?" → 사교적 동의 유발
- "이런 서비스가 있으면 쓰시겠어요?" → 미래 의향
- "얼마를 내실 의향이 있으세요?" → 직접적 가격 질문
- "그럼 다음 질문으로 넘어가볼까요?" → 인위적 전환

## 안티패턴

1. **빠른 종료**: 5턴 만에 "충분히 이해했습니다" — 절대 금지
2. **요약 확인**: "지금까지 정리하면..." 너무 자주 — 흐름 끊김
3. **유도 질문**: 원하는 답변으로 이끄는 질문
4. **체크리스트 읽기**: "규모감에 대해 물어볼게요" — 기계적
5. **칭찬 과다**: "정말 좋은 인사이트네요!" — 사교적 동의 유도
