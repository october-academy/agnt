# Build in Public (BIP) 가이드

완성된 제품을 홍보하는 게 아니라, 만드는 과정 자체를 콘텐츠로 공유하는 전략.

## 왜 BIP인가

- **제품이 없어도 시작 가능** — 아이디어 단계부터 공유
- **팔로워가 고객이 됨** — 과정을 지켜본 사람은 론칭 때 전환율 높음
- **피드백 루프** — 공개적으로 만들면 방향이 틀렸을 때 빨리 알 수 있음
- **신뢰 축적** — "이 사람은 실제로 만들고 있구나"
- **비용 0원** — 광고비 없이 오가닉 도달

BIP는 마케팅 전략이자 검증 도구다. 반응이 오면 방향이 맞다는 신호, 안 오면 메시지나 채널을 바꿔야 한다는 신호.

## Navigator 워크플로우별 BIP 콘텐츠

각 워크플로우를 완료할 때마다 자연스럽게 1개 포스트가 생긴다.

| 워크플로우 | BIP 포스트 소재 | 예시 |
|-----------|----------------|------|
| `/agnt:discover` | 문제 발견 과정 | "이런 문제를 겪는 사람이 있을까? 조사해봤다" |
| `/agnt:interview` | 인터뷰 인사이트 | "3명 인터뷰하고 배운 것: 내 가설이 틀렸다" |
| `/agnt:spec` | SPEC 공개 | "SPEC v1 → v2, 인터뷰 후 이렇게 바뀌었다" |
| `/agnt:landing` | 랜딩 공개 | "랜딩페이지 만들었다. 솔직한 피드백 부탁" |
| `/agnt:channel` | 채널 실험 결과 | "GeekNews vs Threads, 어디서 반응이 더 왔나" |
| `/agnt:analyze` | 수치 공개 | "Day 15: 방문 142명, 폼 응답 8건, 배운 것" |
| `/agnt:retro` | 회고 공유 | "30일 회고: 뭘 만들었고, 뭘 배웠고, 뭘 틀렸나" |

**핵심**: Navigator가 다음 행동을 추천할 때마다 BIP 소재도 함께 생긴다.

## 채널별 BIP 전략

### Threads (한국 BIP 핵심)

한국 인디해커/1인 개발자 커뮤니티가 가장 활발한 채널.

- 체류 시간 알고리즘 → 긴 글이 유리. 300자+ 권장
- 매일 1포스트: 오늘 한 일 + 배운 것 + 내일 할 일
- 스크린샷/수치를 넣으면 저장률 증가
- 링크는 댓글에 (본문 링크 = 도달 억제)
- 다른 빌더 글에 매일 3-5개 댓글 → 네트워크 효과
- 해시태그: #빌드인퍼블릭 #1인개발 #인디해커

**효과**: 팔로워 500명 기준 포스트당 1,000-5,000 노출.

### X (글로벌 BIP 핵심)

글로벌 인디해커 커뮤니티의 메인 채널.

- Premium($8/월) 시 도달 ~10x 증가
- 스레드(연결 트윗) 형식이 단일 트윗보다 효과적
- 영어로 쓰면 글로벌 커뮤니티 도달
- 수치 공개 트윗이 바이럴 확률 높음 ("$0 → $1,000 MRR in 30 days")
- 해시태그: #buildinpublic #indiehacker #solopreneur

**효과**: 팔로워 1,000명 기준 트윗당 2,000-10,000 노출 (Premium).

### LinkedIn (B2B BIP)

B2B/SaaS 타겟이면 X보다 전환율 높음.

- "오늘 XX 문제를 해결하려고 YY를 만들고 있다" — 전문성 기반
- 프로필에 빌더 정체성 명시 ("Building X | Solo founder")
- 긴 포스트(1,000자+) + 줄바꿈 많이 → 알고리즘 선호
- 동료/업계 사람이 바로 타겟 유저가 될 수 있음

### Instagram (비주얼 BIP)

- 릴스: 만드는 과정 타임랩스, 데모 영상
- 스토리: 매일 진행 상황 업데이트
- 포스트에 링크 불가 → 바이오 링크 활용
- 디자인/비주얼 중심 제품에 적합

## BIP 포스트 구조 (범용)

```
[Hook] 한 줄 — 스크롤을 멈추게 하는 문장
[Context] 1-2줄 — 뭘 만들고 있고 왜
[Update] 3-5줄 — 오늘 한 일, 수치, 발견
[Insight] 1-2줄 — 배운 것, 놀란 것
[CTA] 1줄 — 질문 또는 링크 (선택)
```

### 예시: 인터뷰 후 포스트

```
3명 인터뷰하고 나서 내 가설이 완전히 틀렸다는 걸 알았다.

프리랜서 일정 관리가 문제라고 생각했는데,
진짜 고통은 "클라이언트 커뮤니케이션"이었다.
3명 모두 같은 말: "일정은 괜찮은데, 연락이 스트레스"

SPEC을 처음부터 다시 쓰고 있다.
인터뷰 안 했으면 3개월 헛짓할 뻔.

혹시 프리랜서분들, 가장 스트레스 받는 부분이 뭔가요?
```

### 예시: 수치 공개 포스트

```
Day 14 기록.

랜딩페이지 배포한 지 5일.
방문: 142명
폼 응답: 8건 (전환율 5.6%)
가장 많이 온 채널: Threads (62%)

놀란 점: GeekNews에서 온 사람은 적지만 전환율이 15%로 가장 높았다.
개발자 타겟이라 그런 것 같다.

다음 주는 GeekNews에 집중해볼 계획.
```

## BIP 포맷 변주

매일 같은 구조는 지루하다. 돌려가며 써라:

| 포맷 | 언제 | 예시 |
|------|------|------|
| 수치 업데이트 | 주 1-2회 | "Week 2: 방문 300명, 전환 12건" |
| 인사이트/교훈 | 발견 즉시 | "인터뷰에서 배운 3가지" |
| 실패담 | 실패 시 | "이번 주 완전히 삽질했다. 이유:" |
| 질문/투표 | 결정 필요 시 | "A vs B, 어떤 게 나을까요?" |
| 스크린샷 | 진행 시 | 대시보드/코드/디자인 캡처 |
| 비교/분석 | 데이터 축적 시 | "Threads vs GeekNews, 2주 실험 결과" |
| 회고 | 주말/마일스톤 | "Day 30 회고: 뭘 배웠나" |

## BIP 하지 말아야 할 것

- **"좋아요/응원 부탁"** — 가치를 주고 자연스럽게
- **매일 같은 포맷 반복** — 위 변주 테이블 참고
- **허세 수치** — "이번 달 1,000명 방문!" (전환이 0이면 무의미)
- **완성만 공유** — 과정(실패 포함)이 더 흥미롭다
- **여러 채널에 같은 글 복붙** — 채널마다 톤/포맷이 다르다

## BIP 시작 타이밍

```
언제 시작?
├── /agnt:discover 완료 직후 → 첫 포스트 가능
│   "이런 문제를 풀어보려고 한다. 관심 있는 사람?"
├── 채널 선택은?
│   ├── 한국 타겟 → Threads 먼저
│   └── 글로벌 타겟 → X 먼저
└── 빈도는?
    ├── 이상적: 매일 1포스트
    ├── 최소: 주 3회
    └── 일관성이 품질보다 중요
```
