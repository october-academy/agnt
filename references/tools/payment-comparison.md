# 결제 솔루션 비교

1인 개발자/인디해커를 위한 결제 솔루션 비교. 사업자 유무, 예산, 타겟 시장, 플랫폼(웹/앱) 기준.

> 검증 기준: 2026-04-05 기준 각 서비스 공식 pricing / docs / help center 공개 페이지 우선. 계약형 수수료이거나 공개 요금표가 불완전한 경우 숫자를 낮추거나 범위형으로 표기.

## 상세 비교

### TossPayments

**적합**: 한국 시장 타겟, 사업자등록 완료, 월 정기결제 서비스

- **수수료**: 공식 fee page 기준 일반 신용·체크카드 `3.4%`, 가상계좌 `건당 400원`, 가입비/연관리비는 계약 형태에 따라 달라질 수 있음
- **정산**: 공식 fee page 기준 평균 `5일 이내`
- **장점**: 한국 최대 PG. SDK/API 문서 우수. 간편결제(토스페이, 네이버페이, 카카오페이) 통합
- **단점**: 사업자등록 필수. 자동결제(빌링)는 추가 리스크 검토 및 계약 필요
- **정기결제**: 공식 빌링 문서 기준 `카드`와 `계좌이체` 자동결제 지원. 간편결제 자동결제는 미지원
- **연동**: REST API + JavaScript SDK. Next.js/React 예제 공식 제공
- **추천 시나리오**: 한국 유저 대상 SaaS, 온라인 강의, 구독 서비스

#### TossPayments MCP / `llms.txt`

토스페이먼츠는 공식 개발자센터에 `LLMs로 결제 연동하기` 가이드를 제공하고, 공식 블로그에서 2025년 6월 MCP 연동 서버 도입을 발표했다. AI 코딩 도구와 함께 쓰려면 아래 두 가지를 같이 기억하면 된다.

1. 문맥 보강용 `llms.txt`
   - `https://docs.tosspayments.com/llms.txt`
   - 프롬프트에 붙여 넣거나, 에이전트가 먼저 읽게 하면 토스페이먼츠 문서를 더 잘 따라간다.
2. MCP 연결
   - 공식 가이드: `https://docs.tosspayments.com/guides/v2/get-started/llms-guide`
   - 예시 설정:

```json
{
  "mcpServers": {
    "tosspayments-integration-guide": {
      "command": "npx",
      "args": ["-y", "@tosspayments/integration-guide-mcp@latest"]
    }
  }
}
```

공식 가이드가 안내하는 대표 경로:
- Cursor: `~/.cursor/mcp.json`
- VS Code: `.vscode/mcp.json`
- Windsurf: `~/.codeium/windsurf/mcp_config.json`
- Claude Desktop: `claude_desktop_config.json`

### 포트원 (PortOne, 구 아임포트)

**적합**: 여러 PG사를 비교/전환하고 싶을 때, 사업자등록 완료

- **수수료**: 공식 pricing 기준 `월 순거래액 5,000만원 미만 무료`, 이후 포트원 이용료 부과. PG사 수수료는 별도
- **장점**: PG사 추상화 — 토스/이니시스/KCP를 동일 API로. PG 전환 시 코드 변경 최소
- **단점**: 추상화 레이어 추가 = 디버깅 복잡. 포트원 장애 시 전체 결제 불가
- **정기결제**: 지원 (PG사별 차이 있음)
- **추천 시나리오**: 여러 PG사를 테스트하고 싶거나 향후 PG 전환 계획이 있을 때

### 래피드 (Latpeed)

**적합**: 사업자 없이 한국 시장에서 디지털 콘텐츠/템플릿 판매

- **수수료**: 공식 가격 페이지 기준 `Free 1.6%`, `Lite 4.6%`, `Pro 1.6%` (Pro는 연 결제 기준 월 `2.4만원`, 표기상 정상가 월 `3만원`)
- **장점**: 사업자등록 불필요. 크리에이터 특화. 가입 즉시 판매 가능
- **단점**: 해외 카드 미지원. 멤버십(정기결제)은 Pro 전용
- **추천 시나리오**: 템플릿, 가이드, 디지털 에셋 판매. 첫 매출 검증용

### Lemon Squeezy

**적합**: 글로벌 시장 타겟, 사업자 없이 SaaS/디지털 상품 판매

- **수수료**: 공식 pricing 기준 `5% + $0.50/건` 기본, docs 기준 `국제결제 +1.5%`, `PayPal +1.5%`, `구독 +0.5%` 등 일부 추가 수수료 가능
- **장점**: MoR(Merchant of Record) — 세금 신고, VAT 처리 대행. 사업자 불필요. 정기결제 기본 제공. Checkout 페이지 자동 생성
- **단점**: 한국 카드 결제 미지원 (해외 카드만). 수수료 높음. 정산 USD
- **정기결제**: 기본 내장. 가격표 + 체크아웃 자동 생성
- **추천 시나리오**: 글로벌 SaaS, 오픈소스 후원, 디지털 상품. 한국 시장만 타겟이면 부적합

### Polar

**적합**: 오픈소스/개발자 도구 수익화, 글로벌 타겟

- **수수료**: 공식 pricing 기준 `4% + $0.40/건` 기본, `국제카드 +1.5%`, `구독 +0.5%` 추가 수수료 가능
- **장점**: 오픈소스 생태계 통합 (GitHub Sponsors 대안). MoR 서비스. 사업자 불필요
- **단점**: 한국 카드 미지원. payout provider/chargeback 비용은 별도
- **추천 시나리오**: CLI 도구, 라이브러리, 개발자 유틸리티 수익화

### Paddle

**적합**: 글로벌 SaaS 구독, 세금/VAT 자동화가 필요할 때

- **수수료**: 5% + $0.50/건
- **장점**: MoR(Merchant of Record) — 세금 신고, VAT 계산, 환불 전부 대행. 사업자 불필요. 공식 help/docs 기준 한국 로컬 카드와 Naver Pay, Kakao Pay, Samsung Pay, Payco 등 현지 결제수단 지원. Checkout Overlay 자동 생성
- **단점**: 수수료 높음 (5%+). 승인 심사가 까다로움 (앱 완성도 필요). 정산 주기 15일
- **정기결제**: 기본 내장. 프로레이팅, 업/다운그레이드 자동 처리
- **추천 시나리오**: 글로벌 SaaS, 다국가 결제+세금을 한 번에 해결하고 싶을 때. 한국 KRW 현지 결제가 중요한 경우 우선 검토 대상

### RevenueCat

**적합**: 모바일 앱 구독 (iOS/Android), 크로스 플랫폼 인앱결제

- **수수료**: 월 MTR $2,500 미만인 달은 무료. $2,500 이상이 되는 달에는 그 달 MTR 전체의 1% 과금 (앱스토어 15-30% 별도)
- **장점**:
  - 크로스 플랫폼 구독 통합 — iOS(StoreKit), Android(Play Billing), Web(Stripe)을 단일 SDK로 관리
  - Paywalls v2 — 노코드 페이월 빌더, 앱 재심사 없이 원격 변경
  - Experiments — 코드 배포 없이 가격/페이월 A/B 테스트, 조건 충족 시 12개월 LTV 예측 우승안 노출
  - 구독 분석 대시보드 — MRR, Churn, Trial Conversion, Cohort Retention
  - RevenueCat Web Billing — RevenueCat 자체 웹 빌링 엔진, Stripe gateway 사용
  - 공식 사이트 기준 89,000개 앱이 신뢰
- **단점**:
  - Web Billing은 Stripe gateway 기준으로 설명하는 편이 정확하며, 국내 PG 중심 운영과는 결이 다름
  - 규모 커지면 1% 절대 금액 증가 (월 MTR $50K → 약 $500/월)
  - 순수 웹 SaaS에는 오버스펙 (앱이 없으면 Paddle/Lemon Squeezy가 적합)
- **정기결제**: 앱스토어 구독 기본 관리 + Web Billing 구독
- **추천 시나리오**: 모바일 앱에 구독 기능을 넣을 때. 특히 iOS+Android 동시 지원, 가격 A/B 테스트, 구독 분석이 필요할 때

## 결정 가이드

```
웹인가, 앱인가?
├── 모바일 앱 (iOS/Android) → RevenueCat
│   └── 웹에서도 결제 받고 싶다 → RevenueCat Web Billing (Stripe)
└── 웹 서비스 →
    사업자가 있나?
    ├── YES → 한국 시장 타겟?
    │   ├── YES → TossPayments (직접) 또는 포트원 (추상화)
    │   └── NO → Paddle (MoR+세금 자동화) 또는 Lemon Squeezy
    └── NO → 한국 시장 타겟?
        ├── YES → 래피드 (디지털 콘텐츠, 연 2,400만원 이내)
        └── NO → Paddle (한국 카드 지원) 또는 Lemon Squeezy 또는 Polar (개발자 도구)
```

## 1인 개발자 추천

**첫 매출 검증 (웹)**: 래피드 (사업자 불필요, 즉시 시작, 수수료 최저)
**검증 후 본격 운영 (웹)**: TossPayments (한국) 또는 Paddle (글로벌 MoR)
**PG사 비교/전환 계획**: 포트원
**모바일 앱 구독**: RevenueCat (월 MTR $2.5K 미만 무료, 페이월 A/B 테스트)
**글로벌 SaaS + 세금 자동화**: Paddle (MoR 내장, 한국 카드도 지원)
