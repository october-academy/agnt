# 분석 도구 비교

1인 개발자를 위한 제품 분석 도구 비교. 비용, 기능, 프라이버시 기준.

## 한 눈 비교

| 도구 | 무료 티어 | 월 비용 (유료) | 이벤트 한도 (무료) | 셀프호스트 | 세션 리플레이 | A/B 테스트 | GDPR |
|------|-----------|---------------|-------------------|------------|--------------|------------|------|
| PostHog | ✅ | $0 (1M까지) | 월 100만 이벤트 | ✅ | ✅ | ✅ | ✅ |
| Google Analytics 4 | ✅ | $0 | 사실상 무제한 | ❌ | ❌ | ❌ | ⚠️ |
| Amplitude | ✅ | ~$49/월 | 월 1,000만 이벤트 | ❌ | ✅ (유료) | ✅ (유료) | ✅ |

## 상세 비교

### PostHog

**적합**: 제품 분석 + 세션 리플레이 + A/B 테스트를 하나로

- **무료**: 월 100만 이벤트, 1.5만 세션 리플레이, 100만 feature flag 요청
- **장점**: 올인원 (분석 + 리플레이 + 실험 + 설문). 셀프호스트 가능. API 우수. 개발자 친화적
- **단점**: 학습 곡선 있음. 대시보드 UI가 GA4보다 투박. 한국어 미지원
- **연동**: `posthog-js` (3KB gzip), `posthog-node` (서버). Next.js 공식 가이드 있음
- **프라이버시**: 셀프호스트 시 데이터 완전 통제. EU 호스팅 옵션
- **추천 시나리오**: 제품 주도 성장(PLG). 유저 행동 분석이 핵심인 SaaS

### Google Analytics 4 (GA4)

**적합**: 마케팅 퍼널 분석, Google Ads 연동, 비용 0원

- **무료**: 사실상 무제한 (데이터 샘플링 있음)
- **장점**: 완전 무료. Google Ads/Search Console 통합. 표준 마케팅 보고서. 대중적 — 자료 풍부
- **단점**: 이벤트 기반 모델 전환 후 복잡해짐. 세션 리플레이 없음. 데이터 소유권 없음 (Google 서버). GDPR 논란
- **연동**: `gtag.js` 또는 GTM. 스크립트 로딩 ~28KB
- **프라이버시**: 데이터가 Google 서버에 저장. EU에서 GDPR 위반 판결 사례 있음
- **추천 시나리오**: 블로그/랜딩/마케팅 사이트. Google Ads 집행 시 필수

### Amplitude

**적합**: 제품 분석 전문. 유저 코호트 분석, 퍼널, 리텐션

- **무료**: 월 1,000만 이벤트 (매우 넉넉). 코어 분석 기능 무료
- **유료**: Starter $49/월~ (세션 리플레이, 실험, 예측)
- **장점**: 코호트 분석 최강. 퍼널/리텐션 시각화 직관적. 팀 협업 대시보드
- **단점**: 세션 리플레이/A/B 테스트는 유료. 셀프호스트 불가. 1인 개발자에게는 과도한 기능
- **연동**: `@amplitude/analytics-browser` (~36KB)
- **추천 시나리오**: 유저 수 1,000+ 이후 코호트/리텐션 분석. 투자 유치용 데이터

## 결정 가이드

```
유저 수?
├── 0-100명 (초기) → PostHog 무료 (올인원, 충분)
├── 100-1,000명 → PostHog 무료 (여전히 한도 안)
└── 1,000명+ → PostHog 또는 Amplitude (용도에 따라)

무엇을 분석?
├── 유저 행동 + 세션 리플레이 → PostHog
├── 마케팅 채널 + 광고 ROI → GA4
└── 코호트 리텐션 + 퍼널 → Amplitude
```

## 1인 개발자 추천

**시작**: PostHog 무료 (제품 분석 + 세션 리플레이 + feature flag 올인원)
**마케팅 채널 추적 추가**: GA4 병행 (PostHog + GA4 = 무료 올인원)
**유저 1,000+ 이후**: Amplitude 검토 (코호트 분석 필요 시)

**팁**: PostHog은 리버스 프록시로 광고 차단기 우회 가능 (Next.js rewrites 활용). 실측 데이터 정확도가 GA4보다 높아지는 이유.
