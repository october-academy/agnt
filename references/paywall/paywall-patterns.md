# 페이월 카피 패턴

페이월 화면에 들어가는 헤드라인, CTA, 가치 설명, 환불 보증 문구의 공식과 예시.

## 헤드라인 공식

### 공식 1: 결과 약속

> "{시간/노력} 만에 {결과}를 {동사}"

예시:
- "5분 만에 점심 파트너를 찾으세요"
- "하루 10분으로 코드 리뷰 시간을 반으로"
- "3클릭이면 세금 신고 끝"

### 공식 2: 고통 직격

> "{고통}에서 벗어나세요"

예시:
- "혼밥의 민망함, 이제 끝내세요"
- "수동 보고서에 매주 3시간 쓰지 마세요"
- "엑셀 지옥에서 해방되세요"

### 공식 3: 비교 대조

> "{기존 방식} 대신 {새로운 방식}"

예시:
- "엑셀 시트 대신 자동 분석 대시보드"
- "메신저 수동 매칭 대신 실시간 추천"
- "10개 탭 대신 하나의 화면"

## CTA 버튼 공식

### 한국 시장 특화 CTA

| CTA 문구 | 적합 상황 | 심리 기제 |
|----------|-----------|-----------|
| "무료로 시작하기" | 프리미엄 전환 모델 | 리스크 제거 |
| "지금 시작하기" | 일회성 결제 | 긴급성 |
| "{가격}/월로 시작" | 구독. 가격 투명 | 명확한 기대치 |
| "7일 무료 체험" | 고가 구독 | 체험 후 결정 |
| "{N}명이 쓰고 있어요" | 초기 트랙션 있을 때 | 사회적 증거 |
| "다음 {N}명만 할인" | 얼리버드 | 희소성 |

### CTA 작성 규칙

- 동사로 시작 — "시작하기", "받기", "써보기"
- 2-5단어 이내
- 색상: 페이지에서 가장 눈에 띄는 색 (accent color)
- 서브텍스트: CTA 아래에 "카드 정보 불필요" 또는 "언제든 취소 가능" 추가

## 가치 설명 불릿

### 공식: 기능 → 이점 → 감정

```
✅ {기능} — {이점}으로 {감정}
```

예시:
- "✅ 자동 매칭 — 5분 안에 점심 파트너가 잡혀서 안심"
- "✅ 실시간 분석 — 수동 집계 없이 바로 의사결정"
- "✅ 원클릭 배포 — 설정 삽질 없이 바로 런칭"

### 불릿 작성 규칙

- 3-5개 불릿 (7개 초과 시 읽지 않음)
- 기능명만 나열하지 말 것 — "왜 나한테 좋은지"까지
- 가장 강한 불릿을 첫 번째에 배치
- "무제한", "프리미엄", "AI 기반" 같은 형용사 남발 금지

## 환불/보증 문구

### 한국 시장 보증 패턴

| 패턴 | 문구 예시 | 적합 |
|------|-----------|------|
| 기간 보증 | "7일 이내 불만족 시 전액 환불" | 디지털 상품, 구독 |
| 결과 보증 | "30일 사용 후 효과 없으면 환불" | 자신 있을 때만 |
| 조건 없음 | "이유 묻지 않고 환불해드립니다" | 신뢰 구축 |
| 첫 달 보증 | "첫 달 무료, 마음에 안 들면 해지" | 구독 서비스 |

### 환불 문구 작성 규칙

- 명확한 기간 명시 (7일, 14일, 30일)
- "전액" 환불인지 부분 환불인지 명시
- 환불 방법을 간단히 안내 (이메일 한 통이면 됨)
- 법적 필수: 전자상거래법상 디지털 콘텐츠도 7일 청약철회 가능 (단, 사용 시작 후 예외 가능)

## 전환율 벤치마크

| 페이월 유형 | 전환율 평균 | 좋은 수준 |
|------------|------------|-----------|
| 하드 페이월 (사용 불가) | 1-3% | 5%+ |
| 소프트 페이월 (제한 사용) | 3-7% | 10%+ |
| 프리미엄 (핵심은 무료) | 2-5% | 7%+ |
| 체험 후 전환 (7일 무료) | 10-25% | 30%+ |

1인 개발자 첫 제품 기준: **방문자의 2-5%가 결제하면 정상**.

## 페이월 화면 구성 순서

```
1. 헤드라인 (결과 약속)
2. 서브헤드라인 (구체적 수치/시간)
3. 가치 불릿 3-5개
4. 가격 + CTA 버튼
5. 보증/환불 문구
6. 사회적 증거 (있으면)
```
