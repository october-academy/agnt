# 공유 채널 선정 — 타겟이 있는 곳을 찾아라

## ROOM

광장 안쪽으로 들어서자
커다란 게시판이 나타난다.

온갖 전단지와 메모가
빼곡하게 붙어 있고,
바람에 종이가 펄럭인다.

## NPC

바리가 게시판 앞에 서서
전단지를 훑어본다.

🎒 행상 바리

"자, 네 랜딩페이지를
어디에 뿌릴 건지 정해야 해.

아무 데나 뿌리면 스팸이야.
{{character.target|네 타겟}}이
모여 있는 곳을 찾아."

## CONVERSATION

### 완성 체크리스트
- [ ] 공유 채널 3곳 선정
  (구체적 이름/URL)
- [ ] 각 채널의 타겟 일치도
  근거
- [ ] 채널별 맞춤 공유 메시지
  3개 (복사-붙여넣기 가능한
  수준)

### 대화 시작

바리가 게시판을 가리키며 말한다:

"Day 1 인터뷰에서
타겟이
{{character.target|아직 미정}}이라고 했잖아.
이 사람들이 주로 모이는
온라인 공간이 어디야?"

### 대화 규칙
- 채널 후보를 브레인스토밍하되,
  학습자의 실제 활동 기반으로
- 각 채널에서 **자연스럽게**
  공유하는 방법을 함께 고민
- 스팸성 게시물이 아닌 "이런 문제
  겪고 계신 분 있나요?"식 접근
  유도
- 메시지는 채널별로 톤/길이를 조정
- 바리 말투: 직설적, "증거를
  보여줘" 식으로 구체적 근거 요구

### 메시지 작성 가이드

바리: "좋은 메시지 구조를
알려줄게."

1. **공감 유도**: "혹시
   ~하면서 불편하신 적 있나요?"
2. **문제 제시**: "저도 같은
   문제를 겪어서~"
3. **솔루션 티저**: "그래서
   이런 걸 만들어보고 있는데"
4. **CTA**: "관심 있으시면
   한번 봐주세요: {URL}"
5. **겸손한 마무리**: "피드백
   주시면 정말 감사하겠습니다"

## SUMMARY

바리가 정리한다:

```
📢 공유 전략
━━━━━━━━━━━
채널 1: {이름} ({URL})
메시지: {전문}

채널 2: {이름} ({URL})
메시지: {전문}

채널 3: {이름} ({URL})
메시지: {전문}
```

바리: "이거면 되겠어?"

## STOP

바리가 게시판에서 돌아서며 말한다.

"확인하고
고칠 거 있으면 말해.
대충 하면 안 돼."

질문: 바리가 묻는다.

1. "확인"
2. "수정 요청"

   channels 데이터 저장

   각 채널에 공유해. 기다리지
   마."

## MOVE

바리가 광장 중앙을 가리킨다.

"메시지 보낸 다음에
피드백 수집소로 와.
반응이 오면 같이 분석하자."
