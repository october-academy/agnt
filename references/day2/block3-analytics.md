# 성과 분석 + 대시보드 가이드

천막 뒤편 작은 탁자에
수첩과 계산기가 놓여 있다.
벽에 핀으로 꽂힌 차트가 보인다.

## GUIDE

### Day 2 증거 두 축

Day 3 SPEC v1 수정에 필요한 증거는
두 가지다.

1. **추적 링크 클릭 수** (정량)
   — 어떤 채널이 트래픽을 보냈는지
2. **정성 피드백** (정성)
   — 사람들이 뭐라고 했는지

이 두 가지만으로
"이 채널에 집중한다" 또는
"메시지를 바꾼다"를
데이터 기반으로 결정할 수 있다.

### 채널별 클릭 성과 확인

https://agentic30.app/dashboard/links 에 접속하여
추적 링크별 클릭 수를 확인한다.

```text
채널별 성과:
- channel 1 (twitter): XX 클릭
- channel 2 (kakaotalk): XX 클릭
- channel 3 (discord): XX 클릭

가장 반응 좋은 채널:
가장 반응 약한 채널:
```

아직 데이터가 쌓이지 않았어도 괜찮다.
기준선을 잡는 것 자체가 의미 있다.

### 클릭과 피드백 교차 검증

클릭 수(정량)와
피드백(정성)을 함께 본다.

- 클릭은 많은데 피드백이 없다
  → 메시지는 끌지만 검증 CTA가 약하다
- 클릭은 적지만 피드백 질이 높다
  → 채널이 정확하다, 도달만 넓히면 된다
- 둘 다 낮다
  → 메시지와 채널을 모두 재검토

이 교차 분석이
내일 SPEC v1의 수정 근거가 된다.

### 대시보드 사용법

https://agentic30.app/dashboard/links 에서
추적 링크를 직접 관리할 수 있다.

**할 수 있는 것:**

1. **링크 생성**: 새 UTM 추적 링크를 직접 만들 수 있다
2. **링크 수정**: 제목, 채널명 등 메타데이터를 수정할 수 있다
3. **링크 삭제**: 더 이상 필요 없는 링크를 정리할 수 있다
4. **성과 모니터링**: 클릭 수, 채널별 비교를 실시간으로 볼 수 있다

> agnt 과정에서 자동으로 만든 링크도
> 이 대시보드에 모두 표시된다.
>
> CLI에서 MCP `create_utm_link`로 만들어도 되고,
> 대시보드에서 직접 만들어도 된다.
> 두 곳 모두 동일한 링크 데이터를 사용한다.

**앞으로 활용법:**

- 새 채널을 테스트할 때마다 추적 링크를 추가한다
- 매일 대시보드에서 클릭 추이를 확인한다
- 효과 없는 채널의 링크는 정리한다
- Week 1 끝에 채널별 성과를 비교해서 집중할 채널을 고른다

### 플랫폼 랜딩 유저 보너스

agentic30에서 랜딩을 직접 배포한 경우,
MCP `get_landing_analytics`로
더 깊은 분석 데이터를 확인할 수 있다.

- 방문자 수, CTA 클릭률, 폼 제출, 퍼널
- 채널별 전환율 비교

외부 사이트를 검증 채널로 쓰는 경우에는
추적 링크 클릭 수 + 정성 피드백이
핵심 증거가 된다.

## ON_CONFIRM

AskUserQuestion에서
"완료" 또는 "아직 데이터가 없어"를
선택했을 때 ON_CONFIRM을 수행합니다.

1. 대시보드에서 확인한 클릭 데이터를 기록합니다.
   데이터가 없으면 기준선(0)으로 기록합니다.
2. 플랫폼 랜딩 유저인 경우,
   MCP `get_landing_analytics`를 호출하여
   보너스 분석 데이터를 조회합니다. (선택)
3. `d2-analytics` 퀘스트를 제출합니다.
4. state.json에
   `analyticsBaseline`, `channelPerformance`,
   `dashboardChecked`를 저장합니다.
